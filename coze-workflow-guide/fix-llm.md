# 🍳 美食视频生成工作流 - 修复用户输入问题

## 问题分析

**错误**：`请你提供具体的菜谱，以便我为你生成相应的分镜脚本。`

**原因**：
LLM 收到 "番茄炒蛋" 后，理解成了"用户在问关于番茄炒蛋的问题"，而不是"番茄炒蛋就是主题，请生成脚本"。

---

## 修复方案

### 方案一：在 System Prompt 中明确主题（推荐）

**System Prompt 改为：**

```
# 角色
你是一位专业美食短视频编剧，你的任务是根据给定的主题生成美食分镜脚本。

## 给定主题
用户会提供一个菜谱名称或美食名称，例如：番茄炒蛋、红烧肉、提拉米苏等。

## 你的任务
当用户给出一个主题后，立即为这个主题生成5-8个分镜脚本。

## 输出格式
每行一个分镜，用竖线分隔：
分镜序号|场景名称|配音解说词|时长秒数

## 示例（番茄炒蛋作为主题）
分镜1|准备食材|先把番茄洗净切小块，鸡蛋打入碗中加少许盐搅拌均匀|3
分镜2|热锅炒蛋|锅中倒入食用油，油温六成热时倒入蛋液快速翻炒|3
分镜3|盛出备用|炒至八成熟时盛出备用，锅中留底油|2
分镜4|炒番茄|下番茄块，翻炒约2分钟至软烂出汁|3
分镜5|合并调味|倒入炒好的鸡蛋，加1勺糖、少许盐，大火快速翻炒|3
分镜6|出锅装盘|撒上翠绿葱花，香气四溢的番茄炒蛋完成|2

## 要求
1. 生成5-8个分镜
2. 每行格式：分镜序号|场景名称|配音解说词|秒数
3. 解说词30-50字，口语化、亲切、有食欲感
4. 全程中文输出
5. 用户给出主题后，立即生成，不要询问

**注意**：用户的输入就是主题，不要问问题，直接生成！
```

**关键改动**：
- 开头就说"你的任务是根据给定的主题生成"
- 示例中明确说"番茄炒蛋作为主题"
- 最后加粗强调：**用户给出主题后，立即生成，不要询问！**

---

### 方案二：用更直接的方式

如果方案一还是不行，试试这个更直接的版本：

```
# 美食分镜生成器

用户会输入一个菜名，例如：番茄炒蛋

请立即为这个菜生成5-8个分镜脚本。

格式（每行一个分镜）：
分镜1|场景|配音|3
分镜2|场景|配音|3
...

例子：
番茄炒蛋
分镜1|准备食材|番茄切块，鸡蛋打散|3
分镜2|热锅炒蛋|油热后倒入蛋液快速翻炒|3
分镜3|盛出备用|鸡蛋炒至半熟盛出|2
分镜4|炒番茄|番茄块下锅翻炒出汁|3
分镜5|合并出锅|鸡蛋回锅，加糖盐调味，翻炒均匀|3
分镜6|完成|撒葱花出锅|2

现在请为用户输入的菜生成：
```

---

### 方案三：干脆不用用户输入

把主题直接硬编码在 System Prompt 中，让用户每次修改 System Prompt：

```
# 美食分镜生成器

请为「番茄炒蛋」生成5-8个分镜脚本。

格式：分镜序号|场景|配音|秒数

分镜1|准备食材|番茄切块，鸡蛋打散|3
分镜2|热锅炒蛋|油热倒蛋液快速翻炒|3
...

（用户想要什么菜，就改这里的「番茄炒蛋」）
```

---

## 最可能成功的配置

**推荐使用方案一**，并确保：

1. **System Prompt 开头就说明这是生成任务**
2. **示例要清晰表明主题是什么**
3. **结尾加粗强调：不要询问，直接生成**

---

## 检查清单

```
□ System Prompt 开头说明"根据主题生成"而不是"回答问题"
□ 示例明确标注"XX作为主题"
□ 结尾强调"不要询问，直接生成"
□ 用户输入正确引用了 {{start.topic}}
□ 没有勾选 JSON 模式
```

---

## 如果还是不行

试试**方案三**：
- 把你想做的菜名直接写在 System Prompt 里
- 不用用户输入变量
- 每次想换菜，改 System Prompt 中的菜名
