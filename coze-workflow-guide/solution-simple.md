# 🍳 最简方案：硬编码菜谱主题

## 问题根源

Coze 工作流的 LLM 节点对"用户输入"变量的处理方式，可能导致 LLM 把它理解成用户在提问，而不是任务指令。

## 最简单的解决方案

**每次想换菜谱时，直接改 System Prompt 中的菜名即可。**

---

## 配置步骤

### 第一步：创建工作流

名称：`food_video_workflow`（只能用英文）

### 第二步：配置大模型节点（硬编码菜名）

**System Prompt：**

```
# 美食分镜生成器

请为「番茄炒蛋」生成5-8个分镜脚本。

## 输出格式
每行一个分镜，用竖线分隔：
分镜序号|场景名称|配音解说词|时长秒数

## 分镜内容
分镜1|准备食材|先把番茄洗净切成小块，鸡蛋打入碗中加入少许盐搅拌均匀备用|3
分镜2|热锅炒蛋|锅中倒入食用油，油温六成热时倒入蛋液快速翻炒|3
分镜3|盛出备用|将炒至八成熟的鸡蛋盛出备用，锅中留底油|2
分镜4|炒番茄|下番茄块，翻炒约2分钟至软烂出汁|3
分镜5|合并调味|倒入炒好的鸡蛋，加1勺糖、少许盐，大火快速翻炒均匀|3
分镜6|出锅装盘|撒上翠绿葱花，香气四溢的番茄炒蛋完成|2

## 注意
- 生成5-8个分镜
- 每行格式：分镜序号|场景|配音|秒数
- 解说词30-50字，口语化
- 全程中文
```

**不需要添加任何用户输入！直接用硬编码的菜名！**

### 第三步：代码节点

```javascript
const input = params.script || '';
const lines = input.trim().split('\n');
const scenes = [];
for (const line of lines) {
  const parts = line.split('|');
  if (parts.length >= 4) {
    const num = parseInt(parts[0].replace('分镜','').trim());
    if (!isNaN(num)) {
      scenes.push({
        scene_id: num,
        scene_name: parts[1].trim(),
        narration: parts[2].trim(),
        duration: parseInt(parts[3].trim()) || 3
      });
    }
  }
}
return {
  scenes: scenes,
  count: scenes.length
};
```

### 第四步：结束节点

输出 `scenes` 和 `count`

---

## 如何更换菜谱

想换菜谱时：

1. 打开工作流
2. 点击大模型节点
3. 找到 System Prompt
4. 把里面的「番茄炒蛋」改成其他菜名
5. 把「分镜内容」部分改成新菜谱的内容
6. 保存，重新运行

---

## 这个方案的优缺点

| 优点 | 缺点 |
|------|------|
| ✅ 100%有效，不会出错 | ❌ 每次换菜谱要手动改 |
| ✅ 配置简单 | ❌ 不能让用户输入菜名 |
| ✅ 不用担心 LLM 理解问题 | ❌ 不够灵活 |

---

## 如果你想让它更灵活

可以在开始节点添加一个输入变量，然后在 System Prompt 中用占位符：

```
请为「{{topic}}」生成5-8个分镜脚本。
```

但根据你遇到的问题，这可能还是会触发 LLM 的"询问"行为。

**建议：先用硬编码方案，确保能跑通整个流程后，再研究更灵活的方案。**
