## 1. 创建工作流基础设置

### 1.1 登录扣子平台

打开手机或电脑浏览器，访问：

```
https://www.coze.cn/home
```

### 1.2 进入工作流创建页面

```
步骤：
1. 点击左侧菜单「资源库」
2. 点击「工作流」
3. 点击右上角「+ 创建工作流」
```

### 1.3 填写基本信息

```
【工作流名称】
美食视频生成

【描述】
输入菜谱主题，AI自动生成分镜脚本

【确认】按钮
```

## 2. 添加开始节点

### 2.1 认识开始节点

```
开始节点是工作流的入口点，自动存在，不需要手动添加。
它定义了工作流接收的输入参数。
```

### 2.2 配置开始节点的输入变量

```
步骤：
1. 点击画布上的「开始」节点
2. 在右侧设置面板中找到「输入变量」
3. 点击「+ 添加变量」
```

**添加变量 1：**
```
变量名称：topic
变量类型：字符串（string）
描述：菜谱主题，例如「番茄炒蛋」
默认值：番茄炒蛋
必填：是
```

**添加完成后如下图：**

```
┌─────────────────────────────────┐
│  开始节点                        │
│  ────────────                   │
│  📥 输入变量：                    │
│                                 │
│  ┌─────────────────────────┐   │
│  │ topic (字符串) ✓ 必填    │   │
│  │ 描述：菜谱主题            │   │
│  │ 默认值：番茄炒蛋           │   │
│  └─────────────────────────┘   │
│                                 │
│  输出变量：（无需配置）            │
└─────────────────────────────────┘
```

## 3. 添加AI分镜生成节点

### 3.1 拖入大模型节点

```
步骤：
1. 在左侧节点面板找到「大模型」或「LLM」
2. 拖拽到画布上，放在「开始」节点的右侧
3. 节点自动命名为「大模型」
```

### 3.2 配置大模型节点

点击节点，在右侧设置：

```
【节点名称】
AI生成分镜

【模型选择】
点击选择模型 → 滑动选择「豆包 1.5 Pro 32k」
```

### 3.3 配置 System Prompt（最重要！）

```
【System Prompt】（点击编辑区域，复制粘贴以下内容）

# 角色
你是一位专业美食短视频编剧，擅长将家常菜谱转化为生动有趣的分镜脚本。

## 技能
根据用户提供的菜谱主题，生成适合短视频的分镜脚本。

## 输出格式
严格按以下JSON格式输出，不要输出其他内容：
{
  "recipe_name": "菜名",
  "scenes": [
    {
      "scene_id": 1,
      "scene_name": "分镜名称",
      "narration": "配音解说词（30-50字，活泼亲切）",
      "duration": 3
    }
  ],
  "total_duration": 总时长
}

## 要求
1. 生成5-8个分镜
2. 每个分镜配音30-50字
3. 解说要口语化、亲切、有食欲感
4. 包含开头、过程、结尾
5. 全程中文输出
```

### 3.4 配置用户输入

```
【用户输入】
点击「+ 添加输入」，选择：
- 变量名：topic
- 类型：字符串
- 值：{{start.topic}}（引用开始节点的topic）
```

### 3.5 配置输出变量

```
【输出变量】
点击「+ 添加输出」：
- 变量名：script
- 类型：字符串
- 描述：JSON格式的分镜脚本
```

### 3.6 完成后的节点

```
┌─────────────────────────────────────────┐
│  🤖 AI生成分镜                           │
│  ─────────────────────────               │
│  模型：豆包 1.5 Pro 32k                   │
│                                         │
│  📥 输入：                               │
│  · topic ← {{start.topic}}              │
│                                         │
│  📤 输出：                               │
│  · script (字符串)                       │

```

## 4. 添加代码解析节点

### 4.1 拖入代码节点

```
步骤：
1. 左侧找到「代码」节点
2. 拖到画布上，放在「大模型」右侧
```

### 4.2 配置代码节点

```
【节点名称】
解析分镜数据

【运行时】
选择：Node.js
```

### 4.3 编写解析代码

```
【代码区域】
复制粘贴以下代码：

const input = params.script || '{}';
try {
  const data = JSON.parse(input);
  const scenes = data.scenes || [];
  return {
    scenes: scenes,
    count: scenes.length,
    recipe_name: data.recipe_name || ''
  };
} catch(e) {
  return {
    scenes: [],
    count: 0,
    recipe_name: '',
    error: String(e)
  };
}
```

### 4.4 配置输入引用

```
【输入】
点击「+ 添加输入」：
- 变量名：script
- 类型：字符串
- 引用：{{llm_generate.script}}
  （或 {{AI生成分镜.script}}）
```

### 4.5 配置输出变量

```
【输出变量】
点击「+ 添加输出」：
1. 变量名：scenes
   类型：数组
   描述：分镜列表

2. 变量名：count
   类型：数字
   描述：分镜数量

3. 变量名：recipe_name
   类型：字符串
   描述：菜谱名称
```

### 4.6 完成后的节点

```
┌─────────────────────────────────────────┐
│  💻 解析分镜数据                         │
│  ────────────────────────               │
│  运行时：Node.js                        │
│                                         │
│  📥 输入：                               │
│  · script ← {{AI生成分镜.script}}       │
│                                         │
│  📤 输出：                               │
│  · scenes (数组)                        │
│  · count (数字)                         │
│  · recipe_name (字符串)                  │

## 5. 添加结束节点

### 5.1 认识结束节点

```
结束节点是工作流的终点，自动存在。
它定义工作流返回给用户的结果。
```

### 5.2 配置结束节点的输出

```
步骤：
1. 点击画布上的「结束」节点
2. 在右侧找到「输出映射」
3. 点击「+ 添加输出」
```

**添加输出映射：**

```
1. 变量名：scenes
   类型：数组
   值：{{code_parse.scenes}}

2. 变量名：count
   类型：数字
   值：{{code_parse.count}}

3. 变量名：recipe_name
   类型：字符串
   值：{{code_parse.recipe_name}}
```

## 6. 连接所有节点

### 6.1 认识连接点

```
每个节点边缘有小圆点（●），这是连接点。
· 输出连接点：在节点的右侧和底部
· 输入连接点：在节点的左侧和顶部
```

### 6.2 连接步骤

```
连接 1：开始 → AI分镜
1. 点击「开始」节点右侧的 ●
2. 拖动线条到「AI分镜」节点左侧
3. 线条自动吸附，表示连接成功

连接 2：AI分镜 → 解析
1. 点击「AI分镜」节点右侧的 ●
2. 拖动到「解析」节点左侧

连接 3：解析 → 结束
1. 点击「解析」节点右侧的 ●
2. 拖动到「结束」节点
```

### 6.3 连接完成后的工作流

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│         │     │         │     │         │     │         │
│  开始   │────▶│ AI分镜  │────▶│  解析   │────▶│  结束   │
│         │     │         │     │         │     │         │
│  📥    │     │  🤖     │     │  💻     │     │  📤    │
│ topic   │     │ script  │     │ scenes  │     │scenes   │

## 8. （可选）添加剪映小助手

> 如果你已经有「剪映小助手」插件，可以添加以下步骤

### 8.1 添加插件

```
步骤：
1. 左侧点击「插件」
2. 搜索「剪映小助手」或「CapCut Mate」
3. 点击添加插件
4. 拖入画布，放在「解析」节点下方
```

### 8.2 配置剪映API

根据剪映小助手的接口，配置以下功能：

**创建草稿：**
```
API: create_draft
参数：
  - width: 1080
  - height: 1920
  - draft_name: {{code_parse.recipe_name}}_美食视频
```

**添加图片：**
```
API: add_images
参数：
  - draft_url: {{create_draft.draft_url}}
  - image_urls: [AI生成的图片URL列表]
```

**添加字幕：**
```
API: add_texts
参数：
  - draft_url: {{create_draft.draft_url}}
  - texts: {{code_parse.scenes}}
```

## ❓ 常见问题

### Q1: 找不到某个节点？
```
检查左侧面板是否有分类折叠，点击展开即可看到所有节点类型。
```

### Q2: 连接线条无法拖动？
```
确保点击的是节点边缘的小圆点（●），而不是节点本身。
```

### Q3: LLM输出格式不对？
```
检查System Prompt是否准确复制，建议用鼠标选择全部提示词后粘贴。
```

### Q4: 代码节点报错？
```
检查代码语法是否正确，Node.js需要用JavaScript语法。
```

### Q5: 如何添加更多节点？
```
在左侧面板找到需要的节点类型，拖入画布即可。
工作流支持任意数量的节点。
```
